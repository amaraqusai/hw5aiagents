============================================================
ANALYSIS
============================================================
The standard HuggingFace loading mechanism attempts to allocate
the ENTIRE model into CPU RAM in a single operation. For a 14B
parameter model in fp16 (2 bytes per parameter):

  14,000,000,000 params x 2 bytes = 28,000,000,000 bytes = ~28 GB

This far exceeds the 16 GB of physical RAM on this machine.
The operating system's memory manager cannot satisfy this
allocation request, resulting in an immediate OOM crash.

This demonstrates that consumer hardware with 16GB RAM CANNOT
run 14B+ parameter models using standard loading techniques.

SOLUTION: Use AirLLM's layer-by-layer disk streaming approach,
which loads only 1 layer at a time (~700 MB) instead of all 40
layers simultaneously (~28 GB). See airllm_cpu_inference.py.
